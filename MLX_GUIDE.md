# 🍏 Руководство по MLX на Apple Silicon для чайников

Добро пожаловать в мир **MLX**! Если вы привыкли к Ollama и хотите понять, как устроен MLX на вашем Mac и как с ним работать без головной боли — это руководство для вас.

---

## 1. Что такое MLX и почему это магия? ✨

Раньше на обычных компьютерах видеокарта (GPU) и процессор (CPU) имели разную память. Чтобы запустить нейросеть, нужно было сначала загрузить её в оперативную память, а потом «перевозить» данные в память видеокарты. Это как склад и цех в разных зданиях — работает, но логистика съедает кучу времени.

На Mac с процессорами **Apple Silicon (M1, M2, M3, M4)** память **общая (Unified Memory)**. Процессор, графика и нейропроцессор (Neural Engine) «едят из одной тарелки». 

**MLX (Machine Learning Explore)** — это официальная библиотека от команды Apple, написанная специально для того, чтобы локальные нейросети летали на чипах Apple Silicon.
* **Скорость:** MLX обращается к общей памяти напрямую через графический движок Metal, выжимая максимум из вашего графического процессора.
* **Эффективность:** Никакого лишнего перекладывания данных. Меньше задержек, больше токенов в секунду.
* **Лёгкость:** Никаких тяжелых фоновых программ (сервисов-демонов), которые постоянно висят в фоне и едят оперативку.

---

## 2. MLX против Ollama: В чём разница? ⚖️

Если в Ollama всё спрятано под капот, то в MLX всё прозрачно, гибко и открыто:

| Критерий | Ollama | MLX |
| :--- | :--- | :--- |
| **Откуда берутся модели** | Из встроенной библиотеки Ollama. | **Напрямую с Hugging Face** (главный мировой сайт с нейросетями). |
| **Где лежат файлы** | В скрытой папке Ollama (`~/.ollama`). | В стандартном кэше Hugging Face (`~/.cache/huggingface/hub/`). |
| **Фоновые процессы** | Всегда запущен демон Ollama, который ест ресурсы. | Запускается **только внутри приложения** (например, сервера Л.Е.С.) и закрывается вместе с ним. |
| **Имя модели** | `qwen3:14b` | `mlx-community/Qwen3.5-9B-MLX-4bit` |

## 3. Карманный справочник команд (Командная строка) 🛠

Все команды выполняются в терминале вашего Mac. Перед запуском команд рекомендуется активировать виртуальное окружение проекта:
```bash
source .venv/bin/activate
```

### 🔎 Как посмотреть, какие модели уже скачаны?
Все модели MLX лежат в кэше Hugging Face. Чтобы увидеть список загруженных моделей и сколько места они занимают:
```bash
huggingface-cli scan-cache
```

### 🗑 Как удалить модель и освободить место?
Если модель больше не нужна, вы можете удалить её интерактивно (утилита предложит выбрать, что стереть):
```bash
huggingface-cli delete-cache
```

### 📥 Как скачать модель вручную?
Обычно система скачивает модель сама при первом обращении. Но если вы хотите скачать её заранее (например, при хорошем интернете):
```bash
/Users/ovc/Projects/LES_v2/.venv/bin/python3 -c "from huggingface_hub import snapshot_download; snapshot_download('mlx-community/Qwen3.5-9B-MLX-4bit')"
```

### 💬 Как быстро пообщаться с моделью прямо в терминале?
В библиотеку `mlx-lm` встроен очень удобный режим чата для тестов:
```bash
python3 -m mlx_lm.chat --model mlx-community/Qwen3.5-9B-MLX-4bit
```
*(Для выхода из чата нажмите `Ctrl + C`)*

## 4. Настройка локальной модели в Cline (Roo Code) 🤖

Чтобы использовать вашу крутую 9B модель в плагине **Cline** (VS Code), укажите в его настройках следующие параметры:

1. **API Provider**: Выберите `OpenAI Compatible` (или `Custom OpenAI API`).
2. **Base URL**: `http://127.0.0.1:8080/v1` (это адрес нашего локального сервера Л.Е.С. MLX Host).
3. **Model ID**: `mlx-community/Qwen3.5-9B-MLX-4bit` (или просто выберите ее, если подгрузился список).
4. **API Key**: Введите любое слово (например, `local` или `12345`). *Это обязательное требование клиента Cline, хотя наш сервер не проверяет ключи.*

## 5. Как модели управляются внутри Л.Е.С.? 🪵

В нашей локальной экспертной системе управление моделью сделано максимально бережно по отношению к вашей оперативной памяти:

1. **Ленивая загрузка (Lazy Load):** 
   При старте системы веса модели (5-10 ГБ) не загружаются в память. Загружается только легкий токенизатор (~10 МБ). Модель «просыпается» и загружается в память Metal только в тот момент, когда вы отправляете первый вопрос.
2. **Автоматическая выгрузка (TTL - Time to Live):**
   Если вы не общаетесь с Л.Е.С. более 5 минут (300 секунд), сервер автоматически выгружает модель из оперативной памяти Mac. Память освобождается для ваших повседневных задач, браузеров или работы.
3. **Управление через `.env`:**
   Чтобы сменить модель в Л.Е.С., достаточно открыть файл `.env` в корне проекта и изменить параметры:
   ```ini
   LLM_MODEL=mlx-community/Qwen3.5-9B-MLX-4bit
   MLX_MODEL=mlx-community/Qwen3.5-9B-MLX-4bit
   MLX_VAL_MODEL=mlx-community/Qwen3.5-9B-MLX-4bit
   ```
   После изменения файла просто перезапустите службу `mlx` в терминале:
   ```bash
   python3 tools/les_runtime_control.py restart mlx
   ```

Теперь вы полностью контролируете свои локальные нейросети! Приятной работы! 🍏🚀
