# АРТЕЛЬ Фабрика семейств — дизайн пакета

Полный установочный продукт под Windows + Revit. Работает **автономно** (без ЛЕС);
ЛЕС/облако — опциональное обогащение. Один установщик ставит всё; лёгкую модель
качает по выбору.

## Состав пакета (4 компонента + модель)

1. **Морда (Electron)** — десктоп-приложение оператора. Главный интерфейс: задания,
   приём технички, ревью спецификации, запуск генерации, каталог, настройки.
2. **Локальный бэкенд (Python, FastAPI → self-contained exe)** — мозг на машине.
   Несёт **готовое детерминированное ядро**: `artel_datasheet_extractor` (техлист→спец),
   `artel_family_action_plan` (спец→план), `artel_family_geometry` (архетипы),
   `artel_archetype_classifier`. Локальный store (SQLite), ФОП, библиотека архетипов.
3. **Revit-плагин (`ARTEL.Revit.FamilyFactory`, .NET 8)** — исполнитель. Забирает план
   из бэкенда, строит семейство в Revit, шлёт отчёт обратно.
4. **Установщик (Inno Setup, один exe)** — ставит 1–3, опц. качает Ollama + модель.
5. **Лёгкая модель** — только для грязного входа (классификация/маппинг); структурированные
   техлисты идут без модели. Скачивается установщиком по выбору.

## Архитектура — хаб вокруг локального бэкенда

Electron и Revit-плагин **не общаются напрямую** — оба ходят в локальный бэкенд
(`127.0.0.1:5057`). Это снимает связку GUI↔Revit и даёт автономность.

```text
[Electron морда] ──HTTP──┐
                         ├──► [Локальный бэкенд (Python)] ──► локальный store / ФОП / архетипы
[Revit-плагин]  ──HTTP──┘            │
                                     ├─ опц. → лёгкая модель (Ollama, грязный вход)
                                     ├─ опц. → облако (OpenRouter/OpenAI, structured output)
                                     └─ опц. → ЛЕС /api/search (память по проектам)
```

**Почему бэкенд Python, а не C#-порт:** ядро (извлекатель/компилятор/геометрия/классификатор)
уже написано и проверено на Python (KORF MPU извлёкся без модели). Переписывать на C#
= дублировать рабочий код. Revit-плагин остаётся C# (иначе нельзя — Revit API). Морда —
Electron. Так максимум готового идёт в продукт.

## Поток данных (end-to-end)

1. Оператор в **морде** создаёт задание, бросает техничку (PDF/картинка).
2. Морда → бэкенд `/api/extract`: `datasheet_extractor` тащит таблицу → спец (0 модели);
   грязный вход → лёгкая модель заполняет схему `family_spec.v1` (числа из таблицы/OCR).
3. Морда показывает спец (типоразмеры, параметры, ФОП-GUID) → **оператор правит/approve**.
4. Approve → бэкенд компилирует спец+рецепт → `family_action_plan.v1` (детерминированно).
5. Морда «Сгенерировать» → бэкенд кладёт **задание на генерацию** в очередь.
6. **Revit-плагин** (idling-поллинг) забирает задание → строит семейство → сохраняет `.rfa`
   → шлёт `validation report` в бэкенд.
7. Морда показывает отчёт; принято → каталог + learning store (локально), опц. sync в ЛЕС.

Связка Electron↔Revit — через **очередь заданий в бэкенде** (а не прямой вызов): морда
`POST /api/revit/jobs {task_id}`; плагин `GET /api/revit/jobs/next` на Idling, исполняет,
`POST /api/revit/jobs/{id}/report`; морда поллит статус. Revit держится открытым; нет
Revit → морда показывает «открой Revit».

## Компоненты детально

### Морда (Electron)
Экраны: **Задания** (список, новое), **Приём технички** (drag-drop PDF/фото → превью
извлечённой спеки), **Спецификация** (таблица типоразмеров/параметров, правка, approve),
**Генерация** (статус задания в Revit, отчёт по операциям), **Каталог** (принятые
семейства, поиск), **Настройки** (модель/ФОП/ЛЕС-URL/облачные ключи). GUI-first: всё, что
умеет бэкенд, доступно в морде. Electron + локальный HTTP к бэкенду; без облака морда живёт.

### Локальный бэкенд (Python)
FastAPI поверх готовых `tools/artel_*`. Эндпоинты: `/api/extract`, `/api/specs` (CRUD+approve),
`/api/plan` (компиляция), `/api/revit/jobs` (очередь), `/api/catalog`, `/api/quality` (ФОП).
Store: SQLite в `%APPDATA%\ARTEL`. Упаковка: PyInstaller → `artel-backend.exe` (без
системного Python). Прямой доступ к Ollama/облаку/ЛЕС из бэкенда.

### Revit-плагин (.NET 8, готов)
Вкладка ARTEL (Сгенерировать/Проверить/Экспорт). Доработка под пакет: вместо выбора файла
плана — **поллинг очереди бэкенда** (job → план → исполнение → отчёт). Исполнитель планов
(параметры с ФОП-GUID, типы, материалы, многотельная геометрия) — уже есть.

### Лёгкая модель
Нужна **только для грязного входа** (фото/описание/нестандартная вёрстка): классификация
категории/архетипа + маппинг колонок, **structured output** под `family_spec.v1`. Числа —
из таблицы/OCR, не из модели. ≤8 ГБ VRAM (целевой Legion): дефолт **`minicpm-v`** (через
Ollama). Структурированные техлисты (бо́льшая часть) — **без модели вообще**.

## Установщик (Inno Setup, один exe)

`ARTEL-Setup.exe` ставит по выбору:

| Компонент | Обяз. | Действие |
|---|---|---|
| Морда (Electron) | да | распаковка приложения + ярлык |
| Бэкенд (`artel-backend.exe`) | да | служба/автозапуск на `127.0.0.1:5057` |
| Revit-плагин | да | детект Revit 2024/2025 → `.addin`+DLL в Addins |
| ФОП + store | да | путь к ФОП `.txt` + `%APPDATA%\ARTEL` |
| Ollama + модель | **опц.** | скачать OllamaSetup (тихо) + `ollama pull minicpm-v` |
| ЛЕС / облако | опц. | `LES_BASE_URL`, ключи OpenRouter/OpenAI |

Модель и Ollama **качаются установщиком** (не в exe — иначе вес неподъёмный). Детект VRAM →
дефолт модели. Автономность: без интернета ставится ядро (морда+бэкенд+плагин+ФОП), генерация
структурированных техлистов работает офлайн.

## Что готово / что строить

**Готово (проверено офлайн):** детерминированное ядро целиком — извлекатель техлиста,
компилятор плана, геометрия-архетипы, классификатор; Revit-плагин (вкладка, исполнитель
параметров/типов/материалов/геометрии, shared-param с GUID); conformance-kit; реальные
изделия (KORF MPU).

**Строить:**
1. **Бэкенд-обёртка** (FastAPI поверх `tools/artel_*` + очередь заданий + store) — Python, проверяемо на Маке.
2. **Морда (Electron)** — экраны выше; общается с бэкендом.
3. **Плагин → поллинг очереди** (вместо выбора файла).
4. **Лёгкая модель W10.1** (грязный вход) — нужно Ollama/облако.
5. **Коннекторы** (MEP) — тип/размер/грань из техлиста + флаг проверки.
6. **Упаковка**: PyInstaller бэкенда + Electron build + Inno Setup (качает Ollama+модель).

Naming-долг (Agnostis/MyVeras → ARTEL) — закрывается при сборке бэкенда/морды начисто
(старый C# `Agnostis.Api` и `MyVeras.*` — legacy, не путь). См. [[dev-stage-working-style]].
