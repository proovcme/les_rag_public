# Состав MVP

## Название MVP

Задание → спецификация → Revit → отчет.

## Цель

Доказать, что АРТЕЛЬ снижает неопределенность и ручной контроль при разработке Revit-семейств.

MVP не обязан автоматически строить сложную геометрию. На первом этапе важнее закрыть поток работы вокруг задания, параметров, типоразмеров, проверки и готовности к публикации в каталог.

## Сценарий MVP

1. BIM-менеджер создает задание на разработку семейства в веб-сервисе.
2. Пользователь загружает исходники: ТЗ, ФОП/shared parameters, шаблон, таблицу типоразмеров, референсы.
3. AI разбирает материалы и создает черновик спецификации семейства.
4. BIM-менеджер проверяет или редактирует спецификацию.
5. Разработчик открывает задание в Revit-плагине.
6. Плагин добавляет обязательные параметры и создает типы там, где это возможно.
7. Разработчик вручную завершает семейство в Revit.
8. Плагин проверяет открытое семейство по спецификации.
9. Разработчик отправляет RFA и отчет обратно в веб-сервис.
10. BIM-менеджер принимает версию и публикует ее во внутренний каталог.

## Что входит

### Веб-сервис

- список заданий;
- карточка задания;
- загрузка исходников;
- загрузка ФОП/shared parameters;
- черновик AI-спецификации;
- редактируемая спецификация семейства;
- статусы задания;
- загрузка версии семейства;
- просмотр отчета проверки;
- минимальный внутренний каталог.

### Revit-плагин

- вход пользователя;
- список назначенных заданий;
- панель задания;
- показ спецификации;
- применение обязательных параметров;
- создание типов семейства;
- команда проверки;
- команда отправки результата.

### AI

- разбор загруженных документов задания;
- извлечение обязательных параметров;
- извлечение типоразмеров из текста или таблиц;
- сравнение извлеченных параметров с ФОП;
- генерация чеклиста приемки;
- генерация краткого описания для каталога.

## Что не входит

- автоматическое создание сложной геометрии;
- продвинутый solver ограничений;
- полный визуальный diff RFA-файлов;
- внешний marketplace-каталог;
- multi-tenant архитектура для нескольких компаний;
- автоматическая генерация Revit-шаблонов;
- анализ проектной модели целиком.

## Критерии успеха

- Пользователь может создать задание и приложить исходные материалы.
- AI может создать структурированный черновик спецификации из пакета задания.
- Разработчик может открыть задание в Revit.
- Плагин может создать обязательные параметры и строки типов.
- Плагин может проверить открытое семейство по обязательным параметрам и типам.
- Отправленная RFA-версия появляется в веб-сервисе.
- Принятая версия появляется во внутреннем каталоге.
