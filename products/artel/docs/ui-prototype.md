# UI-прототип

## Цель

Первый UI-прототип показывает АРТЕЛЬ как внутренний рабочий инструмент для BIM-команды: реестр заданий, карточка задания, AI-спецификация, проверка семейства и полноценный внутренний каталог.

Прототип расположен в [`app/index.html`](../app/index.html).

## Принципы интерфейса

- Первый экран — рабочая среда, а не лендинг.
- Основная навигация разделяет два равноправных рабочих пространства: задания и каталог.
- Каталог является не вкладкой задания, а отдельным разделом для поиска, скачивания, сравнения версий и запуска задания на обновление.
- Интерфейс плотный, спокойный и утилитарный: меньше маркетинга, больше статусов, таблиц, проверок и действий.
- Любое AI-действие должно выглядеть как проверяемый шаг: разбор, риск, предлагаемое действие, отчет.
- Карточки используются только для отдельных сущностей: задание, метрика, файл, проверка, каталоговая запись.
- Радиусы UI-элементов ограничены 8px, кроме статусных pills.
- Цвета несут смысл: зеленый — готово, синий — в работе, янтарный — внимание, красный — блокер, фиолетовый — AI/action.

## Экраны в прототипе

- Реестр заданий.
- Карточка задания.
- Вкладка исходников.
- Вкладка спецификации.
- Вкладка проверки.
- Связь задания с каталогом.
- Каталог семейств.
- Карточка семейства.
- История версий семейства.
- Правый AI-инспектор.
- Каталог-инспектор.
