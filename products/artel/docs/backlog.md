# Backlog

## Эпик 1. Веб-сервис заданий

- Создать список заданий.
- Создать карточку задания.
- Добавить загрузку исходных файлов.
- Добавить статусы задания.
- Добавить назначение исполнителя.
- Добавить комментарии и историю изменений.

## Эпик 2. AI-разбор задания

- Разбирать текстовое ТЗ.
- Разбирать Excel-таблицу типоразмеров.
- Разбирать TXT-файл shared parameters.
- Формировать черновик спецификации семейства.
- Показывать источники и уверенность по извлеченным данным.
- Давать список уточняющих вопросов.

## Эпик 3. Спецификация семейства

- Хранить целевую категорию Revit.
- Хранить список обязательных параметров.
- Хранить типы и значения параметров по типам.
- Хранить требования к материалам.
- Хранить чеклист приемки.
- Поддержать ручное редактирование спецификации.

## Эпик 4. Revit-плагин

- Авторизовать пользователя.
- Показывать назначенные задания.
- Загружать спецификацию задания.
- Добавлять обязательные параметры.
- Создавать типы семейства.
- Проверять активное семейство.
- Отправлять RFA и отчет в веб-сервис.

## Эпик 5. Проверка семейства

- Проверять наличие обязательных параметров.
- Проверять типы семейства.
- Проверять заполненность обязательных значений.
- Проверять соответствие категории.
- Формировать отчет с ошибками и предупреждениями.
- Генерировать краткое AI-объяснение проблем.

## Эпик 6. Внутренний каталог

- Создать карточку семейства.
- Публиковать принятую версию.
- Хранить историю версий.
- Добавить поиск по имени, категории и тегам.
- Добавить скачивание актуальной RFA-версии.
- Добавить статус устаревшего семейства.

## Эпик 7. Learning Loop

- Сохранять связку задание -> спецификация -> RFA -> проверка -> приемка.
- Помечать принятые семейства как эталонные образцы.
- Назначать архетип семейства.
- Формировать recipe для повторяемых типов семейств.
- Искать похожие принятые семейства для нового задания.
- Показывать типовые ошибки из похожих кейсов.
- Собирать метрики использования каталога.

## Ближайший практический шаг

Следующий практический шаг — выбрать технологический стек backend/frontend и зафиксировать OpenAPI-схему для endpoints из [MVP API Contract](mvp-api-contract.md).

## Delivery-порядок

1. Web Core без AI: задания, файлы, ручная спецификация.
2. Парсер ФОП и модель shared parameters.
3. Revit-плагин: login, список заданий, получение package.
4. Revit-плагин: apply parameters и create types.
5. Validation report и submit RFA.
6. AI-разбор задания.
7. Приемка и каталог.
