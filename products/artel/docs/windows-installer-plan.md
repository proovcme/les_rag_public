# ARTEL Windows installer plan

Цель: **один `ARTEL-Setup.exe`**, который по выбору пользователя ставит весь АРТЕЛЬ —
Revit-аддин, локальный бэкенд, при желании Ollama + локальную модель — и работает
**автономно** (без ЛЕС; ЛЕС/облако — опциональное подключение). См. позиционирование
в [LES_PRODUCT_NOTE.md](../LES_PRODUCT_NOTE.md) и [les-integration.md](les-integration.md).

Опираемся на действующие ARTEL-артефакты, **не на MyVeras (легаси)**:
`ARTEL.Revit.FamilyFactory` (аддин), `backend/Agnostis.Api` (бэкенд),
`build-family-factory-revit.ps1` (сборка/установка аддина).

## Форма поставки: один exe

Генератор инсталлятора — **Inno Setup** (решение оператора 2026-06-14): один
`ARTEL-Setup.exe`, страница выбора компонентов и `[Code]`-секция на Pascal для своей
логики (детект Revit/VRAM, скачивание и тихая установка Ollama, `ollama pull`). Не
тянет .NET-инсталлятор-проект и не наследует MyVeras.

Что лежит внутрь exe (payload), собирается заранее:
- `ARTEL.Revit.FamilyFactory.dll` + `.addin` (через `build-family-factory-revit.ps1`);
- `Agnostis.Api` как **self-contained single-file exe** (`dotnet publish -r win-x64
  --self-contained -p:PublishSingleFile=true`) — без предустановленного .NET;
- дефолтный ФОП-справочник (shared-parameter `.txt`) и стартовый конфиг.

Ollama и модель **не** кладём в exe — скачиваются по выбору на этапе установки
(иначе вес неподъёмный).

## Компоненты (страница выбора)

| Компонент | По умолчанию | Что делает |
|---|---|---|
| **ARTEL Revit add-in** | вкл (обязательный) | детект установленных Revit (2022–2025+), копирует `.addin`+DLL в `%APPDATA%\Autodesk\Revit\Addins\<ver>` для выбранных версий |
| **ARTEL backend** | вкл | `Agnostis.Api.exe` на `127.0.0.1:5057`, автозапуск/служба |
| **ФОП + data dir** | вкл | путь к ФОП `.txt` + `ARTEL_DATA_DIR` (каталог/learning store) |
| **Ollama runtime** | **опц.** | скачать `OllamaSetup.exe`, тихая установка (`/VERYSILENT`); пропустить, если юзер на облаке/ЛЕС |
| **Локальная модель** | **опц.** | `ollama pull <model>` — подбор по железу (ниже) |
| **Подключение к ЛЕС** | опц. | ввод `LES_BASE_URL`; пусто = полностью локально |
| **Облачные ключи** | опц. | OpenRouter/OpenAI ключ для облачного тира |

Обязателен только аддин+бэкенд+ФОП. Детерминированное ядро (компилятор плана,
архетипы) **модели не требует** — Ollama/модель честно опциональны: нужны лишь для
офлайн-vision (техничка→архетип) и черновика спеки.

## Подбор модели по железу (авто, с override)

Установщик детектит VRAM (`nvidia-smi`, иначе WMI/`dxdiag`) и RAM, ставит дефолт,
юзер может сменить. Ядру модель не нужна — это только vision/спека.

| VRAM | Дефолт (vision/техничка) | Лёгкий тир (класс формы) | Заметка |
|---|---|---|---|
| **≤ 8 ГБ** (целевой Legion: 8 ГБ + 32 ГБ RAM) | **`minicpm-v`** (8B, силён по документам/таблицам) — **дефолт** (решение оператора 2026-06-14) | `moondream` (1.8B) | альтернативы: `llama3.2-vision:11b`; `gemma4:12b` работает, но Q4 ~7–8 ГБ → выгрузка в RAM, batch-скорость |
| **12–16 ГБ** | `gemma4:12b` (кандидат оператора, ADR-9) | `moondream` | целиком на GPU |
| **≥ 24 ГБ / нет GPU** | облако GPT-5.5 (OpenRouter/OpenAI) | — | локаль не упирается, либо облачный тир |

Числа из технички берёт **OCR** (PaddleOCR/Tesseract, CPU, ~0 VRAM), не модель.
Точные теги Ollama фиксируются на сборке (`ollama show <tag>` — доступность тегов
меняется); `gemma4:12b` уже валидирован в рантайме оператора.

## Автономность

- Без интернета/ЛЕС/облака установщик ставит аддин+бэкенд+ФОП → генерация работает.
- `LES_BASE_URL` пуст → локальный store; задан → best-effort обогащение из ЛЕС.
- Никаких Docker/WSL на машине БИМ-менеджера: аддин — в процессе Revit, бэкенд —
  нативный exe, Ollama — нативная установка.

## Что доделать (Legion-сессия, нужен dotnet+Revit)

1. Скрипт Inno Setup/NSIS: компоненты, детект Revit, шаг Ollama (download+silent),
   шаг модели (детект VRAM → pull), шаг ЛЕС/облако.
2. `Agnostis.Api` → self-contained single-file publish + автозапуск/служба.
3. Порт детерминированного ядра на C# (против `conformance/`), локальный ФОП/архетип/store.
4. Сборка `ARTEL-Setup.exe` + smoke на машине с Revit без ЛЕС.
