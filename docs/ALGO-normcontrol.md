# Алгоритм: формальный нормоконтроль v1 (NK-01..04)

Низкоуровневый детерминированный formal checker оформления документации (0 LLM). Это **evidence-слой**:
его выводы встраиваются в верхний RAG-led doc-review ([[DOC_REVIEW_GOST_R_21_101_2026_PLAN]]), а не выносят
итоговый вердикт сами. Сверено с кодом 2026-06-27.

## Проверки

| ID | Что проверяет | Как (детерминированно) |
|---|---|---|
| **NK-01** | формат листов по ГОСТ 2.301 | размеры страниц PDF → распознаваемый формат (A4/A3/…); нераспознан → флаг |
| **NK-02** | текстовый слой / сканы | есть ли извлекаемый текст; нет → скан (нужен OCR), не «пусто» |
| **NK-03** | согласованность шифра комплекта | базовый шифр в именах файлов комплекта совпадает; разнобой → issue |
| **NK-04** | ведомость ↔ фактический состав | ведомость рабочих чертежей сверяется с фактическими файлами; недостача/лишнее → issue |

Принцип: что нельзя проверить формально (layout/штамп/смысл) → `manual_required`, не fake pass.

## Где в коде

- Сервис: `proxy/services/normcontrol_service.py` (`check_cipher_consistency` и др.)
- Роутер: `proxy/routers/normcontrol.py` (`/api/normcontrol`)
- Тесты: `tests/test_normcontrol_service.py`
- Переиспользуется верхним слоем: `document_set_model.py` (модель комплекта) и `doc_review_service.py`
  (NK-03/NK-04 как computed-evidence в RAG-led review).

## Граница / связь

- Это **formal v1**. Доменный нормоконтроль по ГОСТ Р 21.101-2026 (rulepack, retrieval, штамп, ПП-87) —
  верхний слой [[DOC_REVIEW_GOST_R_21_101_2026_PLAN]] (Phases 1-5 реализованы поверх этого движка).
- `normcontrol_service` НЕ должен становиться продуктовым «судьёй»: его выводы — один из источников
  evidence, финальный статус — computed + human decision.
- Штамп/основная надпись (D4-002) — отдельный `title_block_extract_service` (текст + OCR за флагом),
  не здесь.
