# Алгоритм: ФГИС ЦС — цена ресурса по коду («Сплит-форма» → lookup)

Детерминированный **точечный поиск сметной цены/индекса по коду ресурса** из ценовой базы
ФГИС ЦС. 0 LLM (ADR-11). Закрывает узкое место `table_query top-k не SQL` для ценовой части
автоценообразования ЛСР: вместо substring-скана 281k строк — exact-match O(1) по ключу.

## Идея

«Сплит-форма» ФГИС ЦС — справочник сметных цен и индексов на 161k–281k строк (регион × квартал).
Доступ к ней — **lookup по точному коду** (`91.05.01-017`, `01.7.15.06-0111`), а не нечёткая
агрегация. Поэтому отдельный сервис, а не путь row-normalizer/Qdrant (281k чанков в вектор — мусор).

## Структура «Сплит-формы» (вскрыта на реальных файлах СПб 2 кв. 2025/2026)

Шапка — строка с ячейкой «Код ресурса» (~18-я), ниже строка нумерации, затем данные. 9 колонок:
код · наименование · ед. · отпускная цена (базовый) · **сметная цена (базовый)** · № группы ·
наименование группы · сметная цена в текущем ур. (часто «-») · **индекс к группе**.

**Текущая цена** = кол.8 если число, иначе `база × индекс` (ресурсно-индексный механизм).
Считается на конвертации → поле `price_current_eff`.

## Шаги

1. **Парсинг** (`parse_split_form`): найти шапку по «Код ресурса», колонки — по подстрокам заголовка
   (порядок: «текущем» раньше «сметная цена», иначе кол.8 перехватит базовую кол.5). Данные — строки
   с валидным кодом (regex `\d{2}[.\d-]{4,}`), числа через `_safe_float` («-»→None).
2. **Конвертация** (`build_price_parquet`): нормализованный Parquet + колонки region/quarter →
   `data/price_base/{name}.parquet`.
3. **Индекс** (`PriceBook`): `{нормализованный_код: строка}` в памяти (281k мелких строк ≈ 1с,
   кешируется `lru_cache`). `normalize_code` снимает буквенный префикс базы (`ФСБЦ-`/`ФССЦ-`).
4. **Lookup**: `lookup(code)` exact; `price(code, method=index|base)`; `search(name)` подстрокой.

## Где в коде

- Сервис: `proxy/services/fgis_price_service.py` (без LLM).
- API: `GET /api/prices/lookup` · `/search` · `/books`, `POST /api/prices/import` (`proxy/routers/prices.py`).
- GUI: «Инструменты» → карточка «ФГИС ЦС — цена по коду ресурса».
- MCP: `les_price_lookup`.
- Тест: `tests/test_fgis_price_service.py`.

## Проверено

На реальной Сплит-форме СПб 2кв2025 цены сходятся с проверенным `Эталон_кровля` до копейки:
краны 622.62×1.39=865.44, гвозди 70296.2×1.3=91385.06, бруски (через снятие `ФСБЦ-`) 22600.59.

## Границы

- Книга цен импортируется отдельно (большой файл, не в репо; `data/price_base/` в gitignore).
- duckdb не нужен для lookup; понадобится на слое агрегаций/джойнов (сумма по разделу, смета×цены).
- Связь с [[kac]]: код не найден в книге → нужен КАЦ (`kac_service.needs_kac`).
