# Алгоритм: КАЦ — конъюнктурный анализ цен

Детерминированное ядро **конъюнктурного анализа цен**: котировки поставщиков (≥3 на материал) →
выбор экономичного → цена в позицию ЛСР. 0 LLM (ADR-11).

## Идея (регламент)

Методика 421/пр: для материалов, которых **нет в ФГИС ЦС**, собирают КП от **≥3 поставщиков на
каждый материал** и берут наиболее экономичный вариант со ссылкой на источник; цена идёт в позицию
ЛСР как неучтённый материал (графа «Н»). См. концепт [[kac]] в `config/domain/smeta_ontology.yaml`.

## Шаги

1. **Группировка** (`analyze_kac`): котировки `{material, supplier, unit, price, source}` →
   по нормализованному наименованию материала (`_norm_material`: lower/trim/ё→е).
2. **Цена** `_price`: парс «2 300»/«2,45»/мусор → float.
3. **Выбор** по стратегии: `min` (экономичный, дефолт) | `median` (ближайшая к медиане).
4. **Достаточность**: `suppliers ≥ min_suppliers` (дефолт 3) → флаг `sufficient`; иначе помечается.
5. **Контроль**: разброс цен (`spread`, `spread_pct`), несовпадение единиц (`unit_mismatch`).
6. **В ЛСР** (`kac_to_lsr_lines`): выбранная цена → линия `{name, unit, price, source, basis:"КАЦ (≥N КП)"}`.
7. **Экспорт** (`to_xlsx`): по материалам строки поставщиков, выбранный помечен ✓, недостаток — предупреждение.

## Мост к ФГИС ЦС

`needs_kac(code)` — нужен ли КАЦ: проверяет, **отсутствует ли код** в книге цен (`fgis_price_service`).
Регламент дословно: КАЦ — для того, чего нет в ФГИС ЦС. См. [[ALGO-fgis-price]].

## Где в коде

- Сервис: `proxy/services/kac_service.py` (без LLM).
- API: `POST /api/kac/analyze` · `/lsr-lines` · `/generate`, `GET /api/kac/needs` · `/download`
  (`proxy/routers/kac.py`).
- GUI: «Инструменты» → карточка «КАЦ» (редактируемая сетка котировок → анализ → выбор + xlsx).
- MCP: `les_kac`. Классификатор: тип `коммерческое_предложение` (КП → вход КАЦ).
- Тест: `tests/test_kac_service.py`.

- Группировка по точному нормализованному наименованию; fuzzy-сопоставление материалов — задел.
