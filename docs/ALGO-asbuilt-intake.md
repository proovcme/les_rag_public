# ALGO — приёмка смонтированного объёма из исполнительных схем (сканов)

Канон логики сервиса `proxy/services/asbuilt_intake_service.py` (+ vision-слой
`proxy/services/asbuilt_ocr.py`). Читать перед правкой. Принцип — ADR-11/LLM-минимализм:
**единственный шаг с моделью — vision-OCR ячеек; всю остальную логику (контекст, числа,
фильтр, дедуп, свод) делает детерминированный код.**

## Зачем

Исполнительные чек-листы/схемы приходят сканами (1 лист, без текстового слоя, таблицы повёрнуты
на 90°). Нужен «сводный смонтированный объём по этажам и системам». Раньше это делалось руками
(рендер → разворот → найти таблицу → кроп → прочитать). Теперь — кнопкой/эндпоинтом ЛЕС.

## Конвейер

```
PDF-скан
  │  parse_asbuilt_filename()         ← детерм. regex: building/system/floor/line/date из имени
  │  render_pdf_to_images()           ← переиспользуем backend.ocr_parser (pypdfium2), DPI=200
  ▼
страница (PIL)
  │  _detect_rotation()               ← дешёвая проба ориентации (даунскейл, угол с макс. строк)
  ▼
страница @ нужный угол
  │  СТРАТЕГИЯ locate (дефолт):
  │    vision_locate_tables()         ← [LLM] уменьшенный лист → bbox таблиц «…смонтированного…»
  │    crop(bbox+pad) нативного разр. → vision_ocr_tables()  ← [LLM] прочитать таблицу ЦЕЛИКОМ
  │  ФОЛБЭК tiles (если locate пусто):
  │    сетка RxC с перекрытием → vision_ocr_tables() по фрагментам → мерж
  ▼
сырые строки JSON {table,no,name,type,unit,qty}
  │  _to_rows() + _num()              ← детерм. нормализация чисел (1003 / 502,5 / «1 003»)
  │  _is_kept()                       ← фильтр: только name≠заголовок-секции и qty>0
  │  _row_key() dedup                 ← снять дубли перекрытия
  ▼
строки объёма (Row)
  │  to_journal() → field_intake_service.create_entry()
  │     zahvatka = floor/system/line (напр. L5/АУПС/ОП), status=pending, doc_id=PDF
  ▼
журнал объёмов  → /api/field/summary (SUM кодом) → xlsx
```

## Почему locate-then-read, а не сплошной OCR / слепой тайлинг

- **Сплошной OCR листа**: маленькая таблица «смонтированного» теряется на даунскейле — модель
  цепляет крупную легенду/план. Неверная таблица.
- **Слепой тайлинг**: границы режут строки (1003→«100,3», колонка «Кол-во» обрезается); перекрытие
  для целостности → один и тот же ряд читается несколькими фрагментами по-разному → дубли с
  расходящимися числами, свод задваивает. (Оставлен как фолбэк, не дефолт.)
- **locate→read**: найти рамку таблицы и прочитать её одним кропом в нативном разрешении — нет
  ни обрезки строк, ни дублей. Повторяет ручной приём (найти → кроп → прочитать).

## Точность и приёмка (status=pending)

OCR ячеек не идеален: ключевые объёмы (метраж кабеля/трубы, счёт извещателей) проходят, но узкая
колонка «Кол-во» и плотные строки дают единичные ошибки. Поэтому строки падают в журнал как
**`status=pending`** — оператор сверяет/правит перед `confirmed` (стройконтроль и так подписывает
«качество монтажа и объёмы»). Числа в своде считает SQL/код, не модель.

## Движок OCR

`local` (дефолт, `OLLAMA_BASE_URL`/gemma4:12b) — приватно, но на больших листах медленно (риск
таймаута). `cloud` (`OPENAI_BASE_URL`/proxyapi, gpt-4.1) по флагу — точнее и быстрее на плотных
повёрнутых таблицах, но исполнительная уходит во внешний OpenAI. Тело запроса — общее
(`build_vlm_ocr_body`, OpenAI-совместимое).

## Ручки (env)

`LES_ASBUILT_OCR_ENGINE`, `LES_ASBUILT_STRATEGY` (locate|tiles), `LES_ASBUILT_DPI`,
`LES_ASBUILT_MAX_SIDE`, `LES_ASBUILT_LOCATE_SIDE`, `LES_ASBUILT_LOCATE_PAD`, `LES_ASBUILT_TILES`,
`LES_ASBUILT_TILE_OVERLAP`, `LES_ASBUILT_OCR_TIMEOUT`, `LES_ASBUILT_CLOUD_MODEL`.

## Точки входа

- CLI: `tools/asbuilt_extract.py "<pdf|папка>" --engine local|cloud --rotate auto|90 --preview|--write [--xlsx out]`
- API: `POST /api/field/extract-asbuilt` (admin; path внутри `LES_EXTERNAL_SOURCE_ROOTS`),
  `write=false` → превью.
- Чат (Совушка вызывает сама, канал `asbuilt`): `proxy/services/asbuilt_chat_service.py` —
  интент «вытащи смонтированный объём из «<путь>»» (+«облаком») → фоновый прогон (`process_path`
  write=pending) + мгновенный ack; результат читается каналом журнала («свод по L5»). Подключён в
  `proxy/routers/chat.py` рядом с task/field; команда-палитра `/исполнительная` (`command_service`).
- Тесты: `tests/test_asbuilt_intake.py` (детерм. части + чат-детектор, OCR/фон замоканы).
