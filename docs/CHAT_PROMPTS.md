# Справочник промтов ЛЕС — как и о чём спрашивать

Что ЛЕС умеет в чате и как сформулировать запрос, чтобы получить точный ответ.
Принцип: **числа и факты считает код (0 LLM), а не выдумывает** — поэтому конкретный запрос
бьёт в детерминированный канал (мгновенно, с обоснованием), а расплывчатый уходит в RAG.

Каналы проверяются по порядку; первый сработавший отвечает. Если ничего не поймало — RAG по документам.

---

## 1. Сметное ценообразование (детерминированно, 0 LLM)

### Глоссарий — «что такое …»
Определение + зачем + из чего выходит → во что + норматив. Из доменной онтологии.
- `что такое КАЦ` · `что такое ВОР` · `что такое ЛСР` · `что такое КС-2`
- `что значит стеснённость` · `расскажи про ГЭСН` · `дай определение ФГИС ЦС`

### Цена ресурса по коду
- `цена 91.05.01-017` · `сколько стоит 01.7.15.06-0111` · `расценка <код>`
- → текущая сметная цена (база × индекс), наименование, единица, регион/квартал.
- Нет кода в базе → подскажет, что нужен КАЦ.

### Нужен ли КАЦ
- `нужен ли КАЦ для <код>` · `<код> есть в ФГИС ЦС?`
- → есть в базе (цена) / нет → КАЦ (≥3 поставщика на материал).

### Коэффициент стеснённости
- `коэффициент стеснённости для города` · `коэф стеснённости действующее предприятие`
- `коэффициент стеснённости памятник культурного наследия` · `какой коэффициент стеснённости` (список)
- → ×ОЗП / ×ЭМ + основание.

> Тяжёлый структурный ввод (таблица 3 КП для КАЦ, многопозиционная сборка ЛСР) — в админке
> **Инструменты** (ФГИС ЦС / КАЦ / Стеснённость / Сборка ЛСР), там форма удобнее.

## 2. Таблицы и сметы — счёт по данным (0 LLM)

Числа берутся из таблиц (Parquet), не из текста. Указывай ПРЕДМЕТ.
- `сумма по ведомости` · `сколько кабеля 3х1,5` · `общая стоимость материалов`
- **Сверка:** `сверь ведомости и акты` (ВОР ↔ КС-2 ↔ смета ↔ ИД: расхождения/пробелы).
- **ВОР:** `сделай ВОР из спецификации` · `переведи спецификацию в ВОР`.

---

## 3. Документы и нормативы (RAG)

Спрашивай КОНКРЕТНО — с термином, шифром или разделом; общий вопрос даёт «нет данных».
- ✅ `требования к огнестойкости кабельных проходок` · `что СП 484 говорит про шлейфы`
- ✅ `серия лотков ДКС в каталоге` (если каталог проиндексирован)
- ❌ `расскажи всё про пожарку` (слишком广 — уйдёт в общий ответ)
- Если «Нет данных в выбранных источниках» — значит документа нет в индексе (загрузи в Самовар).

---

## 4. Объекты и проекты

- `реестр проектов` · `какие объекты` · `общая карта папок` — все объекты + мета из LES.md.
- `пойми папку «/путь/к/папке»` — привязать папку к объекту (контекст как CLAUDE.md).
- `дай сводку проекта` — стадия / ТЭП / состав.

---

## 5. Приёмка и полевые объёмы

- `вытащи смонтированный объём из «/путь/скан.pdf»` (+`облаком` — точнее на плотных таблицах) →
  распознаёт исполнительную/чек-лист → строки в журнал объёмов (pending).
- `запиши объём: <что, сколько, где>` — ручная запись в журнал.

---

## 6. Режимы, команды, память

- `/команды` — палитра slash-команд; `/спецификация //вор //смета //акт //сверка //сводка`.
- `режим облако` / `переключи на локально` / `какой режим` — local / cloud / mix.
- `запомни: <факт>` — заметка; всплывёт в релевантных ответах.
- `помощь` / `что ты умеешь` / `как спрашивать` — этот справочник в чате.

---

## Как формулировать (правила)

1. **Конкретность.** Термин, шифр, код ресурса, путь — а не «расскажи про…».
2. **Числа — у таблиц.** «сумма / сколько / общая стоимость» бьёт в табличный счёт; у текста чисел не проси.
3. **Деньги/нормы — детерминированно.** ЛЕС считает кодом и не выдумывает; «нет данных» = в индексе пусто, а не «не знаю».
4. **Один вопрос — один смысл.** Не склеивай 3 вопроса в строку — разбей.
5. **Пути — в кавычках:** `«/Users/.../папка»`.
