# Проектный граф знаний («второй мозг» проекта)

Дизайн-набросок (2026-06-14, запрос оператора). Цель: не строить generic-Obsidian, а
**сшить уже существующие слои LES в граф вокруг объекта строительства** — решения,
нормативы, BIM-элементы, задачи, объёмы связаны с проектом и друг с другом.

Принципы: **LLM-минимализм** (рёбра выводятся детерминированно — regex/FTS/SQL/ID-матч,
без моделей), **GUI-first** (карта объекта + граф), **максимум переиспользования**.

## Что уже есть (переиспользуем, не переписываем)

| Слой | Где | Статус |
|---|---|---|
| Заметки/решения | `les_notes` (W16.3) | плоские, без связей |
| Рабочая память/recall | W16.1 (лексический) | в контекст чата |
| Задачи | `les_tasks` (W16.2) | без привязки к элементам |
| Объёмы | `les_field_entries` (position/zahvatka/volume/unit/status, W8) | есть захватка |
| База знаний | RAG-датасеты (НТД/почта/CAD-BIM), 169К чанков | есть |
| BIM-элементы | `cad_bim_elements`/`cad_bim_relations` (source_id, object_type) | есть |
| Связи НТД | `GET /api/rag/graph/edges` — упоминания СП/ГОСТ через FTS (W5.7) | есть |
| Граф-вьювер | `qdrant_visualizer` (force-directed, «как Obsidian», W5.7) | рендер-подложка |
| Подсветка элементов | `cad_bim_highlight` (regex Source ID, W6.7) | есть |

**Ключевой пробел:** нет сущности «объект/проект» — датасеты это домены (NTD_FIRE…), а не стройки.
И нет типизированных рёбер между личным слоем (решения/задачи/объёмы) и документами/НТД/BIM.

## Модель (аддитивно, без миграции существующих таблиц)

- `les_projects(id, name, code, address, status, created_at)` — объект строительства.
- `les_project_links(project_id, kind, ref)` — что относится к проекту (`kind`∈{dataset, cad_bim_import,
  note, task, field_zahvatka, mail_thread}). Тег-таблица — не трогаем 5 таблиц колонками.
- `les_edges(id, src_kind, src_id, dst_kind, dst_id, edge_type, source, created_at)` — типизированные
  рёбра. `edge_type`∈{references_ntd, applies_to, mentions_element, located_on, confirms_volume,
  blocks, wiki_link}. `source`∈{wikilink, fts_ntd, bim_id, zahvatka, manual}.

## Откуда берутся рёбра — детерминированно (0 LLM)

1. **`[[вики-ссылки]]`** в тексте заметок/задач — regex; цель резолвится по: заголовку заметки,
   `doc_name`, шифру норматива (СП/ГОСТ), `#N` задачи, `source_id` BIM-элемента.
2. **Ссылки на НТД** — переиспользуем FTS-извлечение из `/api/rag/graph/edges`: заметка/задача/документ
   упоминает СП 4.13130 → ребро `references_ntd`.
3. **Ссылки на BIM** — переиспользуем `cad_bim_highlight.extract_highlight` (regex Source ID): текст
   упоминает элемент → ребро `mentions_element`/`located_on` (этаж из `cad_bim_elements`).
4. **Захватка/позиция** — объёмы ↔ задачи/заметки с той же захваткой/позицией → `confirms_volume`/`applies_to`.

Backfill-джоб строит `les_edges` по существующим данным; инкрементально — при создании заметки/задачи/объёма.

## GUI (payoff)

- **КАРТА ОБЪЕКТА** (dossier): выбрал объект → один экран: нормативы в области, решения/заметки,
  открытые задачи, прогресс объёмов (% по захваткам), сводка BIM (этажи/категории), связанная почта.
  Всё — SQL-джойны + существующие сервисы, без LLM.
- **Граф-наложение**: расширить W5.7 — хаб-объект + личный слой (решения/задачи/элементы) с
  типизированными рёбрами, фильтр по проекту.

## Карта плана — Волна 17 (предложение)

- **W17.1 Сущность объекта** · M — `les_projects` + `les_project_links`; чат-команда «проект: …» /
  GUI-селектор привязки датасета/импорта/задач/объёмов к объекту. Приёмка: создать объект, привязать
  датасет+BIM-импорт+захватки, увидеть состав.
- **W17.2 Типизированные рёбра** · L — `les_edges` + детерминированный вывод (вики-ссылки/FTS-НТД/
  BIM-id/захватка) + backfill. Приёмка: заметка «[[СП 4.13130]] по [[этаж 01]]» даёт 2 ребра; рёбра видны.
- **W17.3 Карта объекта (dossier)** · M `[gui]` — `GET /api/project/{id}/dossier` (нормативы/решения/
  задачи/объёмы-%/BIM/почта, 0 LLM) + экран «КАРТА ОБЪЕКТА». Приёмка: реальный объект → полный паспорт.
- **W17.4 Граф проекта** · M — наложение личного слоя и типизированных рёбер на W5.7, фильтр по объекту.
  Приёмка: объект-хаб с решениями/задачами/НТД вокруг.
- **W17.5 (опц., gated) LLM-синтез** · S `[gate]` — «суммируй решения по объекту», «что не закрыто перед
  сдачей» — ТОЛЬКО после доказанного детерминированного слоя (ADR-11).

## Риск (назвать прямо)
Лёгко скатиться в «свой Obsidian хуже Obsidian». Организующий принцип — **объект строительства**, а не
свободный PKM. Рёбра — типизированные и доменные, не «всё связано со всем». Сила LES в домене — её и усиливаем.
