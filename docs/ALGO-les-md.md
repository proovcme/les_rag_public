# ALGO — LES.md: файл-контекст папки (CLAUDE.md для ЛЕС)

Канон сервиса `proxy/services/les_md_service.py` (+ чат `les_md_chat_service.py`, роутер
`proxy/routers/les_md.py`). 0 LLM на разбор (YAML + regex, ADR-11).

## Зачем

Замыкание инсайта «ЛЕС — это harness»: как Claude Code читает `CLAUDE.md`/`AGENTS.md`, чтобы
понять репозиторий, так **ЛЕС читает `LES.md` (или `ЛЕС.md`), чтобы понять папку**. Кладёшь файл
в папку → ЛЕС знает, что это за объект, к какому проекту относится, какие там типы документов,
что с ними делать и что игнорировать; свободное тело подмешивается в контекст запросов по проекту.

Логика оператора: «даём папку → ЛЕС понимает → работает с ней; папку можно отнести к проекту, но
обращаться можно и вне проекта».

## Формат

```
---
project: Лесной64        # имя объекта/проекта; 'global' → без привязки
object:  Котельная
stage:   РД              # ИД/РД/ПД/ИРД
cipher:  ЭПС-П-01-20-01
address: СПб, Лесной 64
pipelines:               # директивы: тип → что авто-делать
  ид: asbuilt            #   исполнительные сканы → приёмка смонтированного объёма
  спецификации: spec_to_bor
ignore:                  # что не индексировать (glob)
  - '*.bak'
  - '*.dwl*'
---
# свободное тело — Совушка читает его в контекст запросов по этому проекту (как CLAUDE.md)
```

Ключи frontmatter канонизируются RU/EN (`_KEY_ALIASES`): `проект|project`, `объект|object`,
`стадия|stage`, `шифр|код|cipher`, `адрес|address`, `директивы|pipelines`, `игнор|ignore`.

## Конвейер

```
папка
 │ find_les_md()          ← LES.md | ЛЕС.md | les.md… (оба алфавита, регистронезависимо)
 │   нет файла + write_draft → generate_draft() (скан типов/шифров/дат → черновик, оператор правит)
 ▼ parse_les_md()         ← YAML-frontmatter + тело (markdown)
 ▼ read_and_bind()
 │   find-or-create проект (project_service.les_projects) по project|object
 │   link_entity(project_id, "folder", путь)          ← привязка папки к объекту
 │   _store_context(project_id, meta, body)           ← таблица les_md_context
 ▼ context_for_chat(project_id)  → блок в промпт ВСЕГДА для in-project (chat.py рядом с memory_block)
```

«К проекту, но и вне» = двойной режим: `project_id` сужает ретрив к датасетам объекта; без него —
кросс-проектный ретрив. LES.md лишь задаёт привязку (или `global`).

## Авто-инициализация (LES.md руками НЕ пишут — как `/init` для CLAUDE.md)

«Пойми папку «<путь>»» → если LES.md нет, ЛЕС **сам** его собирает заполненным и привязывает.
Детерминированно выводит из скана: `object` (`_infer_object`: снять ведущий «00_», `_`→пробел),
`stage` (`_infer_stages`: стадии из верхнеуровневых подпапок ИРД/ПД/ГЭ/РД/ИД/СДО), `cipher`,
системы (АУПС/ОВ/ТМ/…), `pipelines` (ИД→asbuilt, спец→spec_to_bor), `ignore`. Адрес детерминированно
не вывести → пустой.

**Обогащение (opt-in, «разберись внимательно»):** `enrich_blanks` дочитывает якорные доки
(договор/задание/титул) через `converter` + LLM (`_llm_text`, облако, best-effort) и заполняет
пустые object/stage/address; правки точечно вписываются в frontmatter (`_patch_frontmatter`). 0 LLM
в дефолтном пути — только по явному «внимательно».

## Точки входа

- Чат (Совушка сама): «пойми/разбери папку «<путь>»» (авто-init+привязка), «разберись внимательно …»
  (+LLM-обогащение) — канал `les_md` (`les_md_chat_service`).
- API: `POST /api/les-md/read {path, write_draft}` · `POST /api/les-md/draft {path}` (превью) ·
  `GET /api/les-md/context/{project_id}`. path — внутри `LES_EXTERNAL_SOURCE_ROOTS`.
- Тесты: `tests/test_les_md.py`.

## Состояние / задел

v1: авто-init + парс + привязка (`folder`-link) + контекст + LLM-обогащение + чат живые. Директивы
`pipelines` парсятся и возвращаются — **авто-исполнение** (ИД→asbuilt при индексации) подключается
отдельным шагом (сейчас оператор зовёт явно). Дальше — авто-init **при индексации папки** (без
команды).
