## Единый список фиксов

### P0 — критично (данные/безопасность)
1. **ВОЛК: `admin123` + дефолтный JWT_SECRET активны** — сгенерировать сильные, дописать в .env, ротировать admin-ключ; код: дефолт «admin123» → падать с ошибкой, не молчать.
2. **СУХАРИК: нет restore + бэкап мёртв 10 дней** — реализовать restore (SQLite+Qdrant), починить launchd (USB-права/назначение), checksum, сделать свежий бэкап.
3. **САМОВАР: рассинхрон MetaDB↔Qdrant при partial upsert** — обернуть upsert+статус в одну транзакцию, откат статуса при сбое батча; health-чек после каждого этапа.

### P1 — высоко
4. **КОТ: substring-матчинг без границ слова** (мина маршрутизации) — перевести `route_query` на regex со словограницами (как уже сделано в kot_service).
5. **САМОВАР: срез 6000 симв в классификаторе** → неверная маршрутизация крупных доков; **memory-gate fail-open без MLX** → индексация без лимита (OOM на windows-lite).
6. **ДИАГНОЗ: normalize прячет реальные ошибки** — нормализацию только для UI, не для API/правды; Docker-чек убрать или сделать настоящим.
7. **ВОЛК: X-Forwarded-For спуфинг + 127.0.0.0/8** — TRUSTED_PROXY сузить, loopback через is_loopback, не /8.

### P2 — средне
8. **ТОСКА: переосмыслить как «метку», не гейт** — семантический overlap вместо lexical; единая таблица переходов статусов с тестами; калибровка порогов.
9. **ЕЖИК: дедуп по Message-ID на импорте + OCR вложений**; aioimaplib/OAuth2.
10. **Чистка: мёртвый backend/diagnostics.py; дубль trust-логики (proxy/security.py ↔ sovushka/trust.py) → одна библиотека; хардкод PROXY_URL в backend/auth.py; les_metrics.db не пишется — разобраться.**
