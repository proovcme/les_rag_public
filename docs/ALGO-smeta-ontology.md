# Алгоритм: доменная онтология сметного дела (ЛЕС понимает ВОР/КАЦ/ЛСР/КС)

Чтобы ЛЕС **понимал** предметку — что такое документ/понятие, зачем он, из чего выходит и во что
превращается — знание живёт в редактируемом артефакте + RAG, а **НЕ в весах** модели. 0 LLM.

## Идея

Знание = факты и связи, которые меняются (методики, ставки, редакции) → их место в онтологии/RAG
(редактируемо, цитируемо, точно), а не в fine-tune/LoRA (веса замораживают и галлюцинируют; LoRA —
для формы/поведения, не для фактов). См. [[llm-minimalism-principle]].

Модель: концепт = узел; `inputs`/`outputs` = рёбра графа деривации. Источник истины —
`config/domain/smeta_ontology.yaml` (27 концептов: ТЗ, РД, ВОР, спецификация, ГЭСН, ФГИС ЦС, методы,
НР/СП, КАЦ, коэф. стеснённости, ЛСР, ССР, КС-2/3, АОСР, ОЖР, ИД). id выровнены с типами `doc_classifier`.

## Шаги

1. **Загрузка** (`load_ontology`, кеш): yaml → `{by_id, order}`.
2. **Резолв** (`get_concept`): по id/термину/синониму. Аббревиатура до тире индексируется точно
   («КАЦ»→kac); фолбэк-подстрока берёт **самый короткий ключ** (иначе «кац» матчит «специфиКАЦия»).
3. **Деривация** (`derivation`): BFS по `inputs` (upstream) и `outputs` (downstream) с защитой от циклов.
   Пример: ЛСР ← ВОР+ГЭСН+ФГИС ЦС+КАЦ; ЛСР → КС-2 → КС-3.
4. **Целостность** (`validate`): висячие ссылки рёбер (гейт в тесте).
5. **Рендер**: `glossary_markdown` (→ `docs/smeta_ontology.md`, для RAG и людей), `mermaid_graph` (для GUI).

## Где в коде

- Сервис: `proxy/services/smeta_ontology_service.py`.
- Источник истины: `config/domain/smeta_ontology.yaml`. Глоссарий: `docs/smeta_ontology.md`.
- MCP: `les_glossary` (определение + деривация для внешнего агента).
- Тест: `tests/test_smeta_ontology_service.py`.

## Как используется

- RAG: `docs/smeta_ontology.md` индексируется → чат отвечает «что такое КАЦ» с цитатой.
- Агент/MCP: `les_glossary` отдаёт структурно (что/зачем/из чего→во что).
- Связь: концепты [[kac]]/ФГИС ЦС определяют, что строят сервисы [[ALGO-kac]]/[[ALGO-fgis-price]].

## Границы

- Расширение = правка yaml (новый концепт/ребро — строки, не код).
- Это онтология ДОМЕНА (понятия), не онтология ОБЪЕКТА (`ontology_service`, Floor→System→Category).
