# План улучшения UI «Совушка» (system-design)

Дата: 2026-06-26. Каркас: requirements → high-level → этапы → trade-offs. Цель — убрать
«терминально-хакерскую» эстетику и inline-стиль-суп, сделать читаемо/консистентно/поддерживаемо,
**не ломая живой рантайм** (инкрементально, каждый этап отгружается и проверяется).

## 1. Требования

**Функциональные (не теряем):** чат (стрим, копия, источники, артефакты-таблицы/SVG/mermaid),
админка (Состояние/Датасеты/Визуал/Доступ), настройки (провайдеры/модели/почта), вход, тёмная/светлая тема.

**Нефункциональные:** база шрифта ≥13px (сейчас 9–10px); единый визуальный язык (не 100 inline-стилей);
работа по http/туннелю; тема через токены; поддерживаемость (менять облик в одном месте); сохранить
характер «Совушки» (ирония), но не криптичность.

**Ограничения:** NiceGUI 3.12 + Quasar (стили инлайном или через add_head_html); один разработчик;
**нельзя big-bang** — рантайм живой; RU-интерфейс.

## 2. High-level: где правим
- `sovushka/styles.py` — **единственный источник** дизайн-токенов и классов (сейчас там темы).
- `sovushka/components/header.py` — шапка, табы, диалог настроек (главный рассадник inline-стилей).
- `sovushka/pages/chat.py` — чат (бабблы, инпут, источники).
- `sovushka_ng.py` — каркас страниц.

## 3. Этапы (каждый: dev → `make verify` → deploy → рестарт UI → визуальная проверка → commit)

**Этап 0 — СДЕЛАНО:** Shift+Enter = перенос строки; админка 9 акронимов → 4 чистых вкладки
(Состояние/Датасеты/Визуал/Доступ); кнопка «Копировать» в жесте; NO_DATA-валидатор не врёт.

**Этап 1 — Дизайн-токены и типографика (фундамент).**
- В `styles.py`: блок CSS-классов (`sov-card`, `sov-input`, `sov-btn`, `sov-tab`, `sov-section`,
  `sov-label`) + шкалы: отступы `--sp-1..6`, шрифты `--fs-xs..xl` (база 13–14px), радиусы, бордеры.
- Инжект один раз глобально (`ui.add_head_html`).
- Заменять самые частые inline-`.style()` на классы — **по файлу за раз**, начиная с header.py.
- *Trade-off:* большой find-replace; делаем порционно, каждый файл — отдельная проверка/коммит.

**Этап 2 — Иконки и шапка.** Только material-иконки; убрать эмодзи-кнопки (`🌙 ☀ ⚙ 👑 ↻`).
Шапка: бренд + табы + минимальный ряд контролов; бейдж версии остаётся.

**Этап 3 — Рефактор настроек.** Мега-диалог ([header.py:206-460](../sovushka/components/header.py))
разбить на секции с прогрессивным раскрытием (Провайдер и модели · Почта · Опасная зона) — либо
вкладки внутри диалога, либо отдельная страница настроек.

**Этап 4 — Чат.** Современные бабблы (вопрос/ответ), читаемые source-chip’ы, подсказка
«Shift+Enter — перенос», sticky-инпут, аккуратные состояния загрузки/стрима.

**Этап 5 — Консистентность админ-страниц** (Состояние/Датасеты/Визуал/Доступ) на тех же токенах.

## 4. Trade-offs / решения
- **Инкремент vs переписать:** инкремент (токены → миграция по файлу). Big-bang на живом UI = риск
  «снова всё сломал». Класс-миграция обратима и проверяема пофайлово.
- **Характер vs чистота:** бренд «Совушка»/ирония — оставляем в *тоне ответов*; в *навигации/контролах* —
  чистые имена и иконки. Акронимы (С.А.М.О.В.А.Р. и т.п.) — только если осознанный бренд, не по умолчанию.
- **NiceGUI-ограничение:** часть стилей всё равно инлайн (Quasar props) — токены покрывают 80%, остальное
  оставляем точечно.

## 5. Что ревизить по мере роста
- Светлая тема обязана идти от тех же токенов (сейчас две карты переменных — свести к одной).
- Мобиль/узкий вьюпорт (туннель с телефона) — адаптив после токенов.
- Доступность: контраст, размер кликабельных зон.

## Метрика готовности этапа
«Облик меняется правкой в одном месте (styles.py), база шрифта ≥13px, нет эмодзи-кнопок, настройки
не свалка» — и рантайм не сломан (alignment честен, дым-проверка страниц 200).

# Десктоп-упаковка: Tauri (решение Олега, 2026-06-26)

**Решение:** нативную оболочку делать на **Tauri**, НЕ Electron (в 20-30× легче, системный webview,
кросс-платформа Mac+Win). Electron отклонён: ~150-200 МБ + Node-стек на соло-поддержку ради
маржинального выигрыша.

**Что Tauri оборачивает:** локальный веб-UI Совушки (`:8051`). **Вебка остаётся для ZeroTier** —
оболочка указывает на тот же локальный сервер, веб никуда не девается (доступ `10.195.146.98:8051`
и т.п. живёт параллельно).

**Архитектура:**
```
Tauri-окно (Rust, системный WKWebView/WebView2)
   └─ грузит http://127.0.0.1:8051  ← тот же NiceGUI-UI, что и веб
Python-стек (proxy/MLX/Qdrant) — НЕ бандлится в Tauri:
   • Mac: уже launchd-сервисы → Tauri их не дублирует, подключается;
   • lifecycle: трей-меню (статус health / старт-стоп / «Открыть») — Tauri sidecar или вызов launchctl;
   • авто-апдейт: tauri-updater (подпись).
```

**Жёсткое предусловие:** сперва решить **бандл бэкенда** (Python+MLX+Qdrant на чистую машину —
работа `tools/install_les.py`). Tauri даёт ОКНО, но если ставить нечего — окно бесполезно. Порядок:
1) инсталлятор бэкенда → 2) Tauri-шелл. Не наоборот.

**Этапность упаковки:**
- **A (дёшево, сейчас):** app-mode окно (`--app=…:8051`) + меню-бар апп (статус/старт-стоп/открыть) —
  80% «нативности» без нового стека. Веб для ZeroTier как есть.
- **B (полноценный шелл):** Tauri-обёртка + Python-sidecar/launchd-bridge + трей + авто-апдейт —
  ПОСЛЕ инсталлятора бэкенда. Цель — «релиз на чистой Mac/Win».

**Trade-off:** новый Rust-toolchain (соло) — но bundle крошечный, webview системный, и это
единый путь под Mac+Win-релизы. UI остаётся веб (NiceGUI) — Tauri только шелл, не переписывание.
