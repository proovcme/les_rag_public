# Алгоритм: коэффициент стеснённости → пересчёт позиции ЛСР

Детерминированное применение **коэффициента усложняющих условий (стеснённости)** к позиции ЛСР:
коэф. к ОЗП и ЭМ → пересчёт ФОТ → НР → СП → Всего. 0 LLM (ADR-11).

## Идея (регламент)

Пояснительная записка / Методика 421/пр: при стеснённых/усложняющих условиях к **ОЗП** (оплата
труда рабочих) и **ЭМ** (эксплуатация машин, включая ЗПМ машинистов) применяется повышающий
коэффициент; **материалы не затрагиваются**. Это меняет ФОТ (=ОЗП+ЗПМ), а значит НР и СП (оба от ФОТ).
Концепт [[smeta-ontology]] `coef_stesn`.

## Шаги

1. **Каталог условий** — `config/domain/stesnennost.yaml` (id, label, `k_ozp`, `k_em`, basis).
   Знание ≠ веса: значения типовые, правятся в yaml, сверяются с действующей редакцией норматива.
2. **Расчёт позиции** (`apply_position`): `_calc(ozp, em, zpm, mat, nr%, sp%)`:
   - direct = ОЗП + ЭМ + материалы; ФОТ = ОЗП + ЗПМ; НР = ФОТ×нр%; СП = ФОТ×сп%; Всего = direct+НР+СП.
   - base = `_calc(...)`; adjusted = `_calc(ozp·k_ozp, em·k_em, zpm·k_em, mat, …)`; uplift = Δ Всего.
3. **Применение** (`apply`): по `condition` из каталога ИЛИ явным `k_ozp`/`k_em` (k переопределяет каталог).
   Свод: base_total / adjusted_total / uplift по набору позиций.

## Где в коде

- Сервис: `proxy/services/stesnennost_service.py`; каталог: `config/domain/stesnennost.yaml`.
- API: `GET /api/lsr/stesnennost/conditions`, `POST /api/lsr/stesnennost/apply` (`proxy/routers/lsr.py`).
- GUI: «Инструменты» → карточка «Коэффициент стеснённости» (условие/ручной k + поля позиции → база→скорр).
- MCP: `les_stesnennost`.
- Тест: `tests/test_stesnennost_service.py` (проверяемая арифметика).

## Проверено

Позиция эталона (медный отлив): Всего 12574.45 → 14448.08 (+1873.63) при условии «город» (k=1.15/1.15).

## Границы

- Значения коэффициентов — типовые; точный фактор зависит от проектных условий (сверять редакцию).
- Применяется к одной позиции/набору; интеграция в полный движок сборки ЛСР — следующая веха.
