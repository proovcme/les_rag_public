## 2. Каталог «Каталоги» — почему 0 хитов (корень, не dim)

- Размерности **совпадают** (эмбеддер 1024 = коллекция 1024) — это НЕ рассинхрон
  эмбеддера.
- Реестр (`/api/rag/datasets`) отдаёт `name="Каталоги", docs=58, **dataset_id=None**`.
- В qdrant точки тегируются `dataset_name` / `dataset_id` (пример: `ARTEL_Index` /
  `d09be4e8…`). Точек с `dataset_name="Каталоги"` — **0**.
- Вывод: точки каталога **есть** в индексе (часть 179k), но под другим тегом, а
  реестровая запись «Каталоги» **потеряла привязку `dataset_id`** → фильтр
  «Каталоги» не маппится → 0. Плюс роутер на «лоток ДКС» уводит в `NTD_ELECTRICAL`
  (нормативка), не в каталог.

1. **Плист-дрейф (КРИТ — источник «танцев»).**
   `start_service` переустанавливает репо-плист (cwd=проект) поверх рантайм-плиста.
   - План: (а) шаблоны плистов параметризовать `LES_HOME` вместо хардкода

     существующий рабочий плист (ставить только если отсутствует / по флагу).

2. **Каталог registry↔qdrant рассинхрон.**
   - План: найти реальный `dataset_name/id` точек каталога (scroll по qdrant);
     починить реестровую запись «Каталоги» (вернуть `dataset_id`) или ре-тег/
     переиндекс с правильной привязкой. Затем проверить фильтр и роутинг.

3. **MLX cold-load 34 с.**
   - План: прогрев модели при старте; раздельные main/val модели + TTL
     (см. память `mlx-model-choice-m4`), чтобы не было холодных перезагрузок.

4. **verify-vision ↔ MLX GPU-контеншен.**

   - План: семафор «один GPU-потребитель»; выгружать ollama-модель сразу после
     распознавания; не гонять verify и MLX-генерацию одновременно.

5. **Конфиг-split рантайма.**
   MLX исторически в проектной папке, proxy/sovushka — в `/Users/ovc/LES`.
   - План: всё ядро из единого `LES_HOME=/Users/ovc/LES`; убрать зависимость от
     проектной папки в рантайме.
