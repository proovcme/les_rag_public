# Алгоритм: layout-aware извлечение PDF (порядок чтения · колонки · таблицы → markdown)

Восстановить **структуру** документа до чанкинга: порядок чтения, колонки, кросс-страничные
таблицы → markdown с pipe-таблицами. Цель Ц11 (ADR-5). 0 LLM — геометрия блоков, не модель.
Стыкуется с Ц9 ([[ALGO-table-query]] / `table_appendix_service`): таблицы выходят как markdown-pipe,
поэтому подъём табличных приложений их распознаёт по плотности `|` и поднимает в ретрив.

## Идея

Стандартный экстракт pymupdf теряет порядок (2 колонки читаются «змейкой») и рвёт таблицы по
страницам. `pdf_layout` кластеризует текстовые блоки по X (колонки) → читает сверху-вниз внутри
колонки → таблицы вытаскивает `page.find_tables()` → рендерит как markdown PIPE-таблицы
(`| .. | .. |`). Результат — текст с сохранённой структурой, дальше штатный чанкинг/эмбеддинг.

## Шаги

1. **Колонки:** блоки группируются в X-кластеры; порог зазора — `LES_LAYOUT_COLUMN_GAP_RATIO`
   (доля ширины, дефолт 0.06). Чтение — top-down внутри колонки, колонки слева-направо.
2. **Таблицы:** `find_tables()`; принимаются при ≥ `LES_LAYOUT_MIN_TABLE_ROWS` (дефолт 2) и
   ≥ `LES_LAYOUT_MIN_TABLE_COLS` (дефолт 2); рендер в markdown-pipe.
3. **Подключение:** обёрнуто в `converter._parse_pdf` в try/except — любой сбой → тихий фолбэк
   на штатный pymupdf-путь. Чисто аддитивно: меняется только текст-выход парсера, индекс не трогается.
4. **Индексные проекции:** после штатного Qdrant upsert parse-поток поддерживает `lexical_chunks`
   для того же файла. Поэтому PDF с текстовым слоем попадает не только в dense-векторный поиск, но и
   в lexical/notebook/hybrid слой. Для старого рантайма без FTS можно сделать backfill из Qdrant payloads
   без повторного чтения PDF.

## Стык с Ц9 (подъём табличных приложений)

Таблицы эмитятся как markdown-pipe → `table_appendix_service` узнаёт их по плотности `|`
(`LES_TABLE_APPENDIX_MIN_PIPES`, дефолт 4), добирает в пул кандидатов при табличном интенте
запроса и гарантирует место в окне ретрива. Так «layout → markdown → подъём в RAG» — одна цепочка.

## Флаги (env)

- `LES_LAYOUT_PDF` — главный выключатель (дефолт «on»).
- `LES_LAYOUT_COLUMN_GAP_RATIO` (0.06) — порог зазора между колонками.
- `LES_LAYOUT_MIN_TABLE_ROWS` (2) / `LES_LAYOUT_MIN_TABLE_COLS` (2) — минимальный размер таблицы.

## Где в коде

- `backend/pdf_layout.py` — извлечение; подключение в `backend/converter._parse_pdf` (фолбэк).
- Стык ретрива — `proxy/services/table_appendix_service.py` (см. [[ALGO-table-query]]).

## Граница

- Тяжёлые движки (Docling/MinerU) не подключены — берём pymupdf-геометрию (легче, без доп. весов).
- Сложная вёрстка (вложенные таблицы, объединённые ячейки) распознаётся не всегда → фолбэк.
- Истинные сканы без текстового слоя идут через OCR-ветку `converter._parse_pdf`; layout-aware путь
  работает по текстовым блокам PDF и не заменяет OCR.
