# Ценообразование в строительстве от азов

Дата: 2026-06-26.

Статус: образовательная статья для оператора и контекст для сметного модуля LES.

Важно: это объяснение логики сметного ценообразования простым языком. Для официальной сдачи
сметы, экспертизы и договорной работы нужно сверяться с действующей редакцией методик Минстроя,
ФГИС ЦС, региональными данными и требованиями заказчика.

---

## 1. Зачем вообще нужна смета

Смета отвечает на простой вопрос:

```text
сколько должна стоить строительная работа по принятой методике расчёта
```

Но хорошая смета не является одной цифрой. Это доказуемая цепочка:

```text
проектное решение
  -> объём работ
  -> сметная норма
  -> ресурсы
  -> цены ресурсов
  -> коэффициенты
  -> накладные расходы
  -> сметная прибыль
  -> итоговая стоимость
```

Если цепочка не прослеживается, смета становится слабой. Можно получить красивый итог, но не
сможешь объяснить, откуда он взялся. Для строительства это опасно: стоимость проверяют заказчик,
техзаказчик, экспертиза, подрядчик, бухгалтерия, юристы и иногда суд.

---

## 2. Смета и коммерческое предложение - не одно и то же

Коммерческое предложение обычно говорит:

```text
мы готовы сделать это за такую цену
```

Смета говорит другое:

```text
по нормативной логике эта работа складывается из таких ресурсов и стоит столько
```

Коммерческая цена может быть выше или ниже сметной. Подрядчик может заложить риск, срочность,
логистику, маржу, дефицит людей, сложность объекта. Смета же пытается привести расчёт к единой
проверяемой системе координат.

В идеале смета помогает всем сторонам разговаривать не про ощущения, а про структуру:

```text
объём верный?
норма подходит?
ресурсы раскрыты правильно?
цены актуальны для региона и квартала?
коэффициенты применены обоснованно?
НР и СП посчитаны по нужному виду работ?
что отсутствует и требует КАЦ?
```

## 3. Основные документы сметного расчёта

### ЛСР

ЛСР - локальный сметный расчёт. Это смета на конкретный вид работ или часть объекта.

Примеры:

```text
ЛСР на земляные работы
ЛСР на металлические конструкции
ЛСР на электромонтаж
ЛСР на благоустройство
```

ЛСР обычно содержит строки работ:

```text
номер позиции
шифр нормы
наименование работы
единица измерения
количество
стоимость единицы
общая стоимость
ресурсы / составляющие стоимости
```

### ОСР

ОСР - объектный сметный расчёт. Он собирает несколько локальных смет в стоимость объекта или
части объекта.

Пример:

```text
ЛСР земляные работы
ЛСР фундаменты
ЛСР каркас
ЛСР инженерные системы
  -> ОСР по зданию
```

### ССР

ССР - сводный сметный расчёт. Это верхний документ по стройке. Он собирает объектные сметы,
затраты заказчика, временные здания, проектные работы, экспертизу, резерв и другие главы.

Если совсем коротко:

```text
ЛСР - работа
ОСР - объект
ССР - стройка целиком
```

## 4. Из чего состоит стоимость работы

Для обычной позиции сметы стоимость строится из прямых затрат, накладных расходов и сметной
прибыли.

```text
сметная стоимость позиции = прямые затраты + НР + СП
```

Прямые затраты:

```text
ОЗП - оплата труда рабочих
ЭМ  - эксплуатация машин и механизмов
ОТм - оплата труда машинистов
М   - материалы, изделия, конструкции, оборудование
```

Накладные расходы:

```text
НР - расходы строительной организации на управление, обеспечение производства,
     охрану труда, бытовые условия, инженерно-технический персонал и прочее
```

Сметная прибыль:

```text
СП - нормативная прибыль подрядной организации
```

Для практического расчёта важно понимать базу начисления. Часто НР и СП считаются от ФОТ.

```text
ФОТ = ОЗП + ОТм
НР = ФОТ * норматив НР
СП = ФОТ * норматив СП
```

Это упрощённая схема для понимания. В реальной смете нормативы зависят от вида работ, методики,
условий и выбранной нормативной базы.

---

## 5. Что такое сметная норма

Сметная норма отвечает не на вопрос "сколько стоит", а на вопрос:

```text
какие ресурсы нужны на единицу работы
```

Например норма может сказать:

```text
на 1 тонну монтажа конструкции нужно:
  91.8 чел-ч труда рабочих
  9.5 маш-ч труда машинистов
  9.0 маш-ч автокрана
  0.5 маш-ч бортового автомобиля
  некоторое количество электродов, кислорода, пропана и т.д.
```

То есть норма - это рецепт производства работ. Цена появляется позже, когда к ресурсам
подставляют текущие или базисные цены.

---

## 6. ГЭСН, ФЕР, ТЕР простыми словами

### ГЭСН

ГЭСН - государственные элементные сметные нормы. Они раскрывают работу в ресурсы.

ГЭСН удобны для ресурсной логики:

```text
работа -> ресурсы -> цены ресурсов -> итог
```

### ФЕР

ФЕР - федеральные единичные расценки. В них работа уже выражена в стоимостной логике
федеральной базы.

### ТЕР

ТЕР - территориальные единичные расценки. Это региональная сметно-нормативная база.

В современных расчётах для бюджетного и экспертного контура всё больше внимания уходит в
ресурсную и ресурсно-индексную логику, потому что она ближе к реальному рынку ресурсов.

---

## 7. Почему префикс нормы важен

Нельзя смотреть только на номер.

Пример:

```text
ГЭСН38-01-001-01
ГЭСНм38-01-001-01
```

Визуально номера похожи. Но это разные базы:

```text
ГЭСН  - строительные работы
ГЭСНм - монтаж оборудования
ГЭСНп - пусконаладочные работы
ГЭСНр - ремонтно-строительные работы
```

Если программа склеит такие нормы по голому номеру `38-01-001-01`, она может взять ресурсы
совсем другой работы. Это не мелкая ошибка в копейках, а архитектурная ошибка сметного расчёта.

Правильный ключ нормы должен учитывать тип базы:

```text
norm_key = <base_type>:<bare_norm_code>

ГЭСН:38-01-001-01
ГЭСНм:38-01-001-01
```

## 8. Три метода определения стоимости

В практическом разговоре чаще всего встречаются три подхода:

```text
базисно-индексный метод
ресурсный метод
ресурсно-индексный метод
```

### Базисно-индексный метод

Идея:

```text
берём стоимость в базисном уровне цен
умножаем на индексы пересчёта
получаем текущий уровень цен
```

Плюс: долго использовался, привычен, технологически прост.

Минус: индекс усредняет реальность. Если конкретный материал резко подорожал, а другой почти не
изменился, общий индекс может плохо отражать настоящую картину.

### Ресурсный метод

Идея:

```text
норма раскрывается в ресурсы
каждый ресурс оценивается по текущей цене
сумма ресурсов даёт стоимость
```

Плюс: максимально прозрачно и близко к рынку.

Минус: нужно иметь текущие цены по огромному числу ресурсов. Это сложно поддерживать.

### Ресурсно-индексный метод

РИМ - компромисс между двумя подходами.

Идея:

```text
часть ресурсов оценивается по текущим сметным ценам
часть пересчитывается индексами по группам однородных ресурсов
```

То есть РИМ не просто "умножить всю смету на индекс". Он пытается считать ближе к ресурсам:

```text
работа -> ресурсы -> текущие цены / индексы групп -> итог
```

---

## 9. Ресурсно-индексный метод простыми словами

РИМ говорит:

```text
сначала разложи работу на ресурсы,
потом оцени ресурсы максимально близко к текущему рынку,
а где точной цены нет - применяй установленный индекс к группе ресурсов
```

Например у нас есть работа:

```text
монтаж металлической конструкции, 1 тонна
```

Норма раскрывает её:

```text
труд рабочих
труд машинистов
автокран
бортовой автомобиль
сварочный аппарат
кислород
пропан-бутан
электроды
металл
```

Дальше по каждому ресурсу система спрашивает:

```text
есть ли текущая сметная цена во ФГИС ЦС для региона и квартала?
если есть - берём её
если нет - есть ли индекс к группе однородных ресурсов?
если есть - пересчитываем через индекс
если нет - нужен КАЦ или ручное обоснование
```

В результате стоимость становится не абстрактной, а ресурсной:

```text
цена труда
цена машин
цена материалов
цена оборудования
отдельные недостающие цены
```

---

## 10. Чем РИМ отличается от базисно-индексного метода

Базисно-индексный подход часто выглядит так:

```text
позиция в базисной цене * индекс = текущая цена
```

РИМ ближе к такой логике:

```text
позиция
  -> ресурсы
  -> ресурс 1 * текущая цена
  -> ресурс 2 * текущая цена
  -> ресурс 3 * индекс группы
  -> ...
  -> сумма
```

Главное отличие:

```text
индекс применяется не как грубый слой поверх всей позиции,
а как инструмент пересчёта там, где нет прямой текущей цены ресурса
```

Практически это означает, что РИМ требует более качественных данных:

```text
нужно правильно раскрыть норму
нужно правильно сопоставить ресурс с ценой
нужно знать регион и квартал
нужно не путать единицы измерения
нужно отдельно обрабатывать отсутствующие цены
```

## 11. ФГИС ЦС

ФГИС ЦС - федеральная государственная информационная система ценообразования в строительстве.
Это главный официальный источник для современного сметного контура: нормативы, цены ресурсов,
индексы, справочная информация, материалы по РИМ.

Для сметчика ФГИС ЦС важна потому, что она отвечает на вопросы:

```text
какая цена ресурса действует в регионе?
за какой квартал опубликованы данные?
есть ли ресурс в базе?
есть ли индекс для группы ресурсов?
какой нормативный документ или сборник используется?
```

В машинной системе вроде LES ФГИС ЦС должна быть не "одним числом", а источником evidence:

```text
resource_code
resource_name
unit
region
quarter
price
source
download/import date
```

Цена без источника - слабое место сметы.

---

## 12. Что такое КАЦ

КАЦ - конъюнктурный анализ цен.

Он нужен, когда ресурс есть в работе, но его нельзя нормально оценить из основной базы цен.

Простыми словами:

```text
если официальной цены нет, надо обосновать рыночную цену документами
```

Обычно для этого собирают коммерческие предложения, прайсы, данные поставщиков, открытые
источники. Важно привести всё к сопоставимому виду:

```text
одинаковая единица измерения
НДС / без НДС
доставка / без доставки
регион
дата
условия поставки
минимальная партия
```

Плохой КАЦ:

```text
нашли одну цену в интернете и вставили
```

Хороший КАЦ:

```text
показали несколько источников,
нормализовали единицы,
объяснили выбор цены,
сохранили ссылки и документы
```

## 13. Как смета рождается из проекта

Правильный порядок:

```text
1. Прочитать проектное решение.
2. Получить ведомость объёмов работ или посчитать объёмы по чертежам.
3. Разбить работы на сметные позиции.
4. Подобрать нормы.
5. Проверить применимость норм.
6. Раскрыть нормы в ресурсы.
7. Найти цены ресурсов.
8. Применить коэффициенты.
9. Начислить НР и СП.
10. Сформировать ЛСР.
11. Собрать ОСР и ССР.
12. Проверить трассу и спорные места.
```

Обратный порядок плох:

```text
сначала придумали цену,
потом подобрали норму,
потом подогнали объём
```

Такую смету трудно защищать.

## 14. Подбор нормы

Подбор нормы - одна из самых сложных частей.

Нужно понять:

```text
что именно делается?
это строительство, монтаж, ремонт или пусконаладка?
какой материал?
какая технология?
какая единица измерения нормы?
есть ли условия, отличающие работу от обычной?
```

Например фраза "монтаж столпа" сама по себе недостаточна. Надо понять:

```text
это металлическая конструкция?
это декоративный элемент?
это монтаж оборудования?
есть ли масса?
как крепится?
какие работы входят: изготовление, доставка, установка, сварка, крепёж, отделка?
```

Для автоматической системы правильная стратегия такая:

```text
дать кандидатов норм,
показать почему они подобраны,
показать сомнения,
потребовать подтверждение инженера
```

Плохая стратегия:

```text
молча выбрать первую похожую норму
```

## 15. Единицы измерения

Единицы измерения - частый источник ошибок.

Проект может дать:

```text
кг
шт
м
м2
м3
комплект
```

Норма может быть:

```text
1 т
100 м2
100 м3
1 шт
1000 м3
```

Перед расчётом надо привести объём к единице нормы.

Пример:

```text
в проекте: 250 кг
норма: 1 т
количество для нормы: 0.25
```

Если забыть это преобразование, ошибка будет в 1000 раз.

---

## 16. Коэффициенты

Коэффициенты применяются, когда условия производства работ отличаются от нормальных.

Примеры:

```text
стеснённость
работа в действующем здании
работа на высоте
зимнее удорожание
сложная логистика
работы в охранной зоне
ночные работы
```

Коэффициент нельзя применять потому, что "объект сложный". Нужно основание:

```text
пункт методики
условия в ПОС/ППР
указание заказчика
проектное ограничение
акт/исходные данные
```

Для LES коэффициент должен быть evidence-объектом:

```json
{
  "coefficient": 1.15,
  "reason": "стеснённые условия",
  "source_ref": "ПОС.pdf#page=12",
  "applies_to": "labor|machines|position|section"
}
```

## 17. НР и СП

НР и СП не надо воспринимать как произвольную надбавку.

Это нормативные начисления:

```text
НР - накладные расходы
СП - сметная прибыль
```

Обычно они зависят от вида работ. Для одной работы норматив может быть один, для другой -
другой. Поэтому при расчёте важно знать не только код нормы, но и вид работ, к которому она
относится.

Практическая проверка:

```text
правильно ли определён вид работ?
правильно ли выбран норматив НР?
правильно ли выбран норматив СП?
от какой базы они начислены?
не применены ли они дважды?
```

---

## 18. НДС, доставка, заготовительно-складские, оборудование

На итоговую стоимость влияют не только работы.

Нужно отдельно понимать:

```text
цена с НДС или без НДС
учтена ли доставка
есть ли заготовительно-складские расходы
это материал или оборудование
кто поставляет: заказчик или подрядчик
входит ли монтаж в цену оборудования
```

Одна и та же строка "изделие X" может считаться по-разному в зависимости от договорной схемы.

---

## 19. Как читать строку ЛСР

Упрощённая строка:

```text
ГЭСНм38-01-001-01
Листовые конструкции массой свыше 0,5 т, сборка с помощью крана
Ед. изм.: т
Кол-во: 1.25
```

Что надо проверить:

```text
код нормы соответствует работе?
единица "т" совпадает с объёмом?
количество 1.25 получено из проекта?
ресурсы раскрылись из правильной базы ГЭСНм?
цены взяты для нужного региона и квартала?
нет missing по материалам и машинам?
НР/СП соответствуют виду работ?
```

Если все ответы имеют source_ref, строка защищаемая. Если половина ответов "ну вроде", строка слабая.

## 20. Что должна делать автоматическая система

Хороший сметный модуль не должен "угадывать смету". Он должен собирать проверяемую цепочку.

Правильный workflow:

```text
ТЗ / проект / ВОР / спецификация
  -> извлечение работ и объёмов
  -> подбор кандидатов норм
  -> подтверждение нормы
  -> раскрытие нормы в ресурсы
  -> поиск цен ресурсов
  -> КАЦ для missing
  -> расчёт кодом
  -> flags/blockers
  -> CSV/XLSX/JSON
```

Главный принцип:

```text
модель связывает и объясняет,
код считает,
источники доказывают,
инженер подтверждает спорное
```

## 21. Что LES должен считать сам, а что нет

LES может и должен считать кодом:

```text
пересчёт объёма в единицу нормы
qty ресурса = расход на единицу * объём
стоимость ресурса = qty * price
ОЗП / ЭМ / ОТм / М
ФОТ
НР
СП
итог позиции
итог раздела
экспорт CSV/XLSX
```

LES не должен самоуверенно решать без evidence:

```text
какая норма единственно правильная
можно ли применять коэффициент
достаточен ли КАЦ
входит ли спорная работа в состав другой позиции
соответствует ли смета всем условиям договора
```

Там нужен статус:

```text
manual_required
missing_evidence
needs_engineer_decision
```

---

## 22. Типовые ошибки

### Ошибка 1. Считать цену без объёма

Нельзя нормально считать без количества.

```text
плохо: сделать смету на крепление
лучше: крепление бронзы, масса 0.42 т, монтаж по ТЗ, регион СПб, 2 кв. 2026
```

### Ошибка 2. Путать похожие нормы

Особенно опасны одинаковые номера в разных базах:

```text
ГЭСН
ГЭСНм
ГЭСНп
ГЭСНр
ФЕР
ТЕР
```

### Ошибка 3. Не проверять единицу нормы

`100 м2` и `1 м2` - не одно и то же. `кг` и `т` - не одно и то же.

### Ошибка 4. Брать цену без региона и квартала

Цена ресурса без региона и периода почти бесполезна.

### Ошибка 5. Молчаливые missing

Если материал не найден, нельзя считать его нулём. Это должен быть blocker.

### Ошибка 6. Прятать КАЦ

КАЦ должен быть видимым: источники, даты, единицы, нормализация.

### Ошибка 7. Доверять итоговой цифре без трассы

Итоговая сумма важна, но для проверки важнее путь к ней.

## 23. Мини-пример расчёта

Дано:

```text
работа: монтаж металлической конструкции
норма: ГЭСНм38-01-001-01
единица нормы: т
объём: 1.25 т
```

Норма даёт ресурсы на 1 т:

```text
труд рабочих: 91.8 чел-ч
автокран: 9.0 маш-ч
бортовой автомобиль: 0.5 маш-ч
сварочный аппарат: 28.6 маш-ч
материалы: кислород, пропан-бутан, электроды и т.д.
```

Умножаем на объём:

```text
труд рабочих = 91.8 * 1.25 = 114.75 чел-ч
автокран = 9.0 * 1.25 = 11.25 маш-ч
бортовой автомобиль = 0.5 * 1.25 = 0.625 маш-ч
```

Дальше:

```text
каждый ресурс * цена ресурса
сумма по труду -> ОЗП
сумма по машинам -> ЭМ
труд машинистов -> ОТм
материалы -> М
ФОТ = ОЗП + ОТм
НР = ФОТ * норматив НР
СП = ФОТ * норматив СП
итог = ОЗП + ЭМ + ОТм + М + НР + СП
```

Если цена электрода не найдена:

```text
не ставить 0
не придумывать цену
поставить missing_price
запустить КАЦ
```

## 24. Какой должна быть выдача LES

Не только:

```text
Итого: 123 456 руб.
```

А так:

```text
Позиция:
  код: ГЭСНм38-01-001-01
  объём: 1.25 т
  источник объёма: ТЗ_столп_22_06.docx#...
  источник нормы: ФГИС ЦС / ГЭСНм, source_guid=...

Ресурсы:
  1-100-40, труд рабочих, 114.75 чел-ч, цена ..., источник ...
  91.05.05-015, автокран, 11.25 маш-ч, цена ..., источник ...
  ...

Начисления:
  ФОТ: ...
  НР: ...
  СП: ...

Blockers:
  missing_price: ...
  manual_required: ...
```

Такой расчёт можно проверять, спорить, исправлять и защищать.

## 26. Главный вывод

Смета - это не таблица и не итоговая сумма. Смета - это доказательная модель стоимости.

Хорошая смета отвечает на вопросы:

```text
что строим?
сколько строим?
по какой норме?
какие ресурсы нужны?
почём каждый ресурс?
почему применены эти коэффициенты?
какие начисления сделаны?
где источники?
что не найдено?
что должен подтвердить инженер?
```

РИМ делает эту логику более ресурсной и более требовательной к данным. Он ближе к реальности,
но хуже переносит хаос: неправильный код, пропущенная единица, цена без региона или missing,
замаскированный под ноль, сразу портят расчёт.

Для LES правильная роль такая:

```text
не угадывать смету,
а собирать проверяемую цепочку от проектного объёма до цены ресурса и итогового XLSX.
```
